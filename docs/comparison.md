# Comparison

Side-by-side technical comparisons between Livemark and the major docs tools. Each page covers stack, content surface, feature parity, and output model — no recommendations or "pick X if" guidance. Draw your own conclusions.

## Tools

:::card{title="Docusaurus vs Livemark" href="/comparison/docusaurus/" icon="book-text"}
React + MDX docs framework from Meta, Webpack-based.
:::

:::card{title="VitePress vs Livemark" href="/comparison/vitepress/" icon="zap"}
Vite-native, Markdown-first, Vue-based docs framework.
:::

:::card{title="Mintlify vs Livemark" href="/comparison/mintlify/" icon="sparkles"}
Commercial hosted SaaS with MDX content and built-in AI.
:::

:::card{title="Starlight vs Livemark" href="/comparison/starlight/" icon="star"}
Astro-based docs theme with islands architecture and MDX.
:::

:::card{title="Fumadocs vs Livemark" href="/comparison/fumadocs/" icon="square-dashed"}
Next.js App Router docs framework with modular core + UI packages.
:::

:::card{title="Nextra vs Livemark" href="/comparison/nextra/" icon="square-stack"}
Next.js docs framework with theme packages and MDX content.
:::

:::card{title="MkDocs vs Livemark" href="/comparison/mkdocs/" icon="book-marked"}
Python-based docs generator; Material is the de-facto theme.
:::

:::card{title="Hugo vs Livemark" href="/comparison/hugo/" icon="feather"}
Go single-binary general-purpose static site generator.
:::

## What every comparison covers

- **At a glance** — language, framework, build tool, content format, license, output model.
- **Architecture** — how each tool is built and what it depends on.
- **Content surface** — Markdown/MDX support, frontmatter, components, includes.
- **Feature matrix** — versioning, i18n, search, themes, plugins, sitemap, RSS, custom routes.
- **Output & deployment** — what each emits and where it can run.
